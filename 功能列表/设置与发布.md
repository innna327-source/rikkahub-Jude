# 设置与发布

## 能力

- 主题、UI、通知、模型、供应商、搜索、语音、MCP、Web 和文件设置。
- 用量提醒与超时锁定：超过设定的每日使用时长后，应用锁和系统锁会显示超限时长，并在次日 00:00 自动解锁。
- GitHub Latest Release 检查、更新卡片和 DownloadManager 下载 APK。
- debug ABI 分包：arm64、x86_64、universal。

## 核心入口

- 设置模型：`data/datastore/PreferencesStore.kt`
- 设置 UI：`ui/pages/setting/`
- 更新检查：`utils/UpdateChecker.kt`
- 更新 UI：`ui/components/ui/UpdateCard.kt`
- 版本：`app/build.gradle.kts`
- 打包：`scripts/build-universal-debug-apk.ps1`

## 发布规则

- 版本判断使用 `versionName`，新 Release 必须高于当前版本。
- Release 资产优先名为 `app-universal-debug.apk`，缺失时才回退 arm64。
- 当前 Git remote 为 `rikkahub-Jude`；源码中的旧仓库更新 URL 仍依赖 GitHub 重定向，后续版本应改为新地址。
- 默认不构建、不提交、不发布；只有用户明确要求才执行。
