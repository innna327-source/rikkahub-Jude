# 项目规则

## 信息优先级

1. 当前源码与 Gradle 配置。
2. `AGENTS.md` 的操作约定。
3. 本知识结构的定位说明。
4. README；README 中仓库地址、发布包说明和编码可能滞后。

## 特殊架构规则

- `app` 是组合根：业务编排、Android UI、Room、DataStore 和系统服务都在此装配。
- `ai` 保持供应商无关，不反向依赖 `app`；应用专属提示词、工具和 MCP 放在 `app/data/ai/`。
- Compose 页面通过 VM/Repository/Service 改状态；跨领域流程进入 Service，不在 Composable 中直接访问 DAO 或模型 API。
- 会话不是平铺消息列表，而是 `Conversation + MessageNode` 分支树；修改删除、重生成和跳转时必须同时考虑当前分支。
- 压缩后的历史由 `compressedSummary` 与 `compressedMessageNodeIds` 协同表示；写回前走 `normalizeCompressionState()`，生成请求不能再次携带已压缩节点。
- 全局配置属于 `SettingsStore`/DataStore；可查询、有关联关系的业务记录属于 Room；图片和附件属于应用私有文件并由 `FilesManager` 管理引用。
- Room 当前版本为 25。新增/修改表必须同步 Entity、DAO、`AppDatabase`、Migration 和 schema。
- 朋友圈按解析后的 scope ID 隔离；普通助手使用助手 ID，有独立系统提示词的会话可使用会话作用域。不要跨 scope 查询或删除。
- 本地工具由 `LocalTools.getTools()` 按助手配置和请求模式注入；语音通话等特殊请求模式可能有额外限制。
- `web-ui` 构建产物复制进 `web/src/main/resources/static`；Android 的 `web:preBuild` 会触发前端构建。

## 技术栈约束

- Kotlin 2.3、JVM 17、Android Gradle Plugin 9.2。
- Android：minSdk 26、target/compileSdk 37。
- UI：Jetpack Compose、Material 3、Navigation 3。
- 状态与异步：ViewModel、Flow/StateFlow、Kotlin Coroutines。
- 注入：Koin，模块位于 `app/.../di/`。
- 数据：Room 2.8、DataStore、kotlinx.serialization。
- 网络：OkHttp 5、Ktor；AI 流式协议包含 SSE。
- Web：React 19、React Router 7、TypeScript、Vite/Tailwind。

## 禁止事项

- 未经明确要求，不提交、不推送、不打 APK、不创建 Release。
- 不把无关工作区改动带进当前修改；开始和结束都检查 `git status --short`。
- 不使用 `git reset --hard`、`git checkout --` 等破坏性命令清理用户改动。
- 不绕过 Repository 直接让 UI 写 DAO；不在多个页面重复实现同一业务状态机。
- 不用字符串拼接替代已有 JSON/Room/serialization 结构。
- 不修改数据库结构却遗漏迁移与 schema；不随意改变已序列化字段名。
- 不把供应商专属判断扩散到 UI；应在 provider/adapter 边界处理。
- 不把大规模上游同步、翻译更新或无关重构混入功能修复。
- 不提交 APK、构建目录、密钥、`local.properties` 或临时目录。

## 核心数据流

### 聊天

`ChatPage -> ChatVM -> ChatService -> GenerationHandler -> ProviderManager -> Provider`

返回流：模型增量输出进入 `ChatService` 会话状态，完成后由 `ConversationRepository` 写入 Room；工具调用可进入 LocalTools、MCP、搜索或记忆仓库。

### 设置

`Setting/Assistant UI -> SettingsStore.update -> DataStore -> settingsFlow -> 各 VM/Service`

助手列表也序列化在 Settings 中；会话、记忆、朋友圈等独立记录不放入 Settings。

### 朋友圈

`MomentsOverlay -> MomentsVM -> MomentRepository -> MomentDAO/Room`

普通聊天中的 `post_moment`/`delete_moment` 由 `LocalTools` 进入同一 Repository。到期的 AI 点赞/评论在打开或刷新朋友圈时由 `MomentsVM` 惰性处理。

### 备份

`BackupPage/VM -> ExportSerializer/Sync -> Room + Settings JSON + FilesManager 文件`

恢复必须兼容旧/新数据库和可空设置字段，并修复本地文件 URI。

### Web

`web-ui -> Ktor API(WebServerManager/ChatService) -> 与 Android 共用会话和设置`

## 模块依赖关系

```text
common <- ai <- search
common <- speech
document/highlight/material3/usage-tracker/weather/web  (独立能力)
以上全部 <- app
web-ui --构建静态资源--> web <- app
```

禁止能力模块依赖 `app`，否则会形成反向依赖并让业务边界失效。

## 验证与交付

- Kotlin 改动默认验证：`.\gradlew :app:compileDebugKotlin`
- 高风险共享逻辑：补跑 `.\gradlew :app:testDebugUnitTest`
- 仅用户明确要求打包时：`powershell -ExecutionPolicy Bypass -File scripts\build-universal-debug-apk.ps1`
- 默认交付 APK：`app/build/outputs/apk/debug/app-universal-debug.apk`
- 明确要求提交时，只暂存相关文件；提交成功后推送当前分支。
