# 项目地图

> AI 首次进入仓库时先读本文件，再按任务跳转到 `功能列表/`；只有定位到具体入口后才读源码。

## 30 秒结论

- 产品：RikkaHub 的非官方 Android LLM 客户端 fork，核心增量是自动上下文压缩，并持续加入用量管理、语音、朋友圈等本地能力。
- 主体：`app` 是业务编排和 Android UI；`ai` 是模型、供应商和消息协议底座；其余 Gradle 模块是可复用能力模块。
- 状态：偏好配置存 DataStore，业务实体存 Room，本地附件由 `FilesManager` 管理，网络模型请求经 `ProviderManager` 分发。
- 入口：应用初始化看 `RikkaHubApp.kt`，页面导航看 `RouteActivity.kt`，聊天主流程看 `ChatService.kt`，依赖装配看 `app/.../di/`。

## L0：产品系统

`RikkaHub Android App`

- 面向多供应商 LLM 的本地聊天客户端。
- 支持助手、记忆、MCP、本地工具、搜索、多模态、TTS/ASR、Web 访问和本地备份。
- fork 特有维护线：滚动摘要、独立压缩/OCR API、用量提醒与应用锁、朋友圈、语音通话交互。

## L1：运行边界

| 边界 | 职责 | 主入口 |
|---|---|---|
| Android 应用 | Compose UI、导航、系统服务、权限、通知 | `app/` |
| AI 运行时 | 模型定义、供应商适配、流式协议、工具协议 | `ai/` |
| 本地数据 | Room、DataStore、私有文件、导入导出 | `app/.../data/` |
| Web 端 | Android 内嵌 Ktor 服务与 React 客户端 | `web/`、`web-ui/` |
| 能力模块 | 搜索、语音、文档、渲染、统计、天气 | 同名 Gradle 模块 |
| 发布运维 | universal/arm64 APK 脚本、GitHub Release 更新 | `scripts/`、`UpdateChecker.kt` |

## L2：代码分层

| 层 | 目录 | 规则 |
|---|---|---|
| 展示层 | `app/.../ui/` | Compose 页面与组件；页面状态进入对应 VM，不直接拼模型请求。 |
| 编排层 | `app/.../service/` | 跨仓库/模型/工具的长流程；聊天核心由 `ChatService` 统一编排。 |
| 领域与持久化 | `app/.../data/repository/`、`data/db/` | Repository 暴露领域操作；DAO/Entity 只处理存储。 |
| 配置层 | `app/.../data/datastore/` | 全局设置、助手配置、供应商、语音、Web、用量规则。 |
| AI 适配层 | `ai/.../provider/`、`app/.../data/ai/` | `ai` 定义通用协议；`app` 组装提示词、转换器、本地工具和 MCP。 |
| 平台能力 | `service/`、各能力模块 | Android 前台服务、文件、语音、搜索、天气和 Web 服务。 |

## Gradle 模块职责

| 模块 | 核心职责 | 依赖方向 |
|---|---|---|
| `app` | 产品业务、Compose UI、Room/DataStore、系统服务 | 依赖全部能力模块 |
| `ai` | AI 消息、Tool、模型和 OpenAI/Claude/Google 适配 | 仅依赖 `common` |
| `common` | Android/HTTP/缓存公共工具 | 基础模块 |
| `search` | 多搜索服务统一接口 | 依赖 `ai` |
| `speech` | ASR/TTS 配置、供应商、播放控制 | 依赖 `common` |
| `document` | PDF/DOCX/PPTX/EPUB 文档解析 | 独立能力 |
| `highlight` | 代码高亮 | 独立能力 |
| `material3` | 项目内 Material 组件扩展 | 独立 UI 能力 |
| `usage-tracker` | Android 用量读取、统计模型和页面 | 独立平台能力 |
| `weather` | 定位天气仓库 | 独立平台能力 |
| `web` | Ktor 服务端及静态资源承载 | 构建时依赖 `web-ui` 产物 |
